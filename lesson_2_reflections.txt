How is the staging area different from the working directory and the repository? What value do you think it offers?
	Stagging area is more like a buffer zone between commit and repository. It gives you option of first getting satisfied with the logical change you want to make and then updating the logical as commit in repository.

How can you use the staging area to make sure you have one commit per logical change?
	All the changes will be stored in the staging area, and as soon as I feel that I have made a logical change I can commit the change.

What are some situations when branches would be helpful in keeping your history organized? How would branches help?
	If I want to keep the basic architecture same and then want different functions to be done then branching would be really helpful.

How do the diagrams help you visualize the branch structure?
	Diagram helps in visualization of flow of commits, thus making branches easier to understand.

What is the result of merging two branches together? Why do we represent it in the diagram the way we do?
	By merging two branch together we are in a way blending the two sets of code to a single set of code. We represent it in the diagram the way we do as it makes life easy for lets say any new coder who didn't have prior knowledge of the project. He/She will get hold of the work done instantly.
